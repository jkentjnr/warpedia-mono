Warpedia design system

Visual and interaction system for warpedia.org — an open, primary-source-rich, encyclopedic resource for the human record of war. The pages below are interactive prototypes built from the project's specification documents. No production code exists yet; the site has not launched.

Site mock-up — warpedia.org

Static page-level designs for the public catalogue, applying the design system to every template called out in the website mock-up guide — Conflict, Event, Person, Place, Equipment, Theme, and the People hub index. The home, Browse, and About surfaces are also drawn. None of the data is live; names and counts come from the sample-data document.

Reference

The design system derives from the project's specification documents. They live in the same repository and are best read on GitHub.

  • Design system README — brand context, content fundamentals, visual foundations, iconography.
  • SKILL.md — the working agreement; what to lift, what to avoid.
  • Project charter — what Warpedia is and is not. The anchor.
  • Data model — Item, Person, Conflict, Event, Unit, Place, Collection, Equipment, Theme, Trial, Treaty.
  • Contribution flow specification — source for the 12-screen contribute prototype.
  • Dublin Core application profile · taxonomies
  • Storage architecture
  • Research — landscape, source-licensing, manuals, content dimensions.
